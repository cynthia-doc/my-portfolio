Tech Corner

Malloc

An implementation of malloc using a segmented list structure with twelve size classes.
The first size class is a non-circular single linked list of mini blocks of size 16 (minimum block size), which only have a header and a pointer to the next free mini block.
The rest of the size classes are non-circular double linked lists with regular blocks, which has a header and a union of a footer and a struct containing the two pointers for the double linked list.
We use LIFO for insertion policy when freeing blocks. And blocks are coalesced right after it is freed and before it is inserted.

When allocating blocks, we use a modified best fit policy, which examines a maximum of 80 blocks, to find a free block.
This is a footerless implementation, so when a block is allocated, it no longer has a footer.

Shell

An implementation of a tiny shell with job control.
The shell has four builtin commands (quit, jobs, fg job, bg job) and runs non-builtin commands as child processes. Only one foreground job runs at a time, and all zombie child processes are reaped right away.
Signals are forwarded to the entire process group that contains the foreground job.
The shell supports I/O redirection for files and builtin command 'jobs' output. But the shell does not support piping.

Proxy

A simple computer program that acts as an intermediary between clients making requests to access resources and the servers that satisfy those requests by serving content. It only handles HTTP/1.0 GET requests. It supports multiple concurrent requests and caches web resources with corresponding url.

Bias in the Entertainment Industry

A study on the bias in the entertainment industry that looks at data from IMDb and predict the likelihood of different types of actors and actresses playing certain genres of film.
To build the model that predicts the likelihood of a celebrity being featured in a certain genre, Naive Bayes classifier with categorical predictors based on region, gender, and peak age is used.
The model is then used to train the training set (80% of the collected data) and predict on the test set (20% of the collected data).
